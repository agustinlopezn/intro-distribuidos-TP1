¿Qué pasa si el archivo pesa 0 bytes?

A corregir:
  - Contemplar que el archivo no exista
  - Si el server no esta levantado y el cliente quiere hacer una accion -> no pasa nada. manda el up,down y se queda esperando.
  - El tiempo del server tiene que ir al thread (sino es instantaneo)
  - El server no imprime el NO_SQ_ACK
  - Si se muere el cliente el server no se entera
  - El server no imprime el progreso por el CHUNK_SIZE grande, capaz moverlo al socket?

GBN:
  - Cada vez que se termina de mandar un chunk, se manda un END. El problema es que el ACK de ese END se manda una sola vez. 

El puerto del cliente se HARDCODEA para poder simular pérdida de paquetes con iptables.

No es necesario que el seq_number de GBN sea de 4 bytes xq se reinicia cada vez que envía un nuevo chunk. Mientras se cumpla:
   techo(CHUNK_SIZE / PACKET_SIZE) < 255, un byte debería alcanzar
   
Falta descartar archivos que excedan el tamaño máximo (4 GB en principio)



Hipotesis: 
- En el server no hay ningun archivo mas alla de los subidos por clientes (No es necesario chequear el tamaño del archivo cuando un cliente descarga)
